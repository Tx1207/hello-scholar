# AMP®-Parkinson's Disease Progression Prediction（2023）竞赛笔记

## 竞赛简介

这是一场面向帕金森病进展预测的表格数据竞赛，核心任务是利用蛋白质、肽段和临床访问信息预测患者未来的 MDS-UPDRS 评分。

### 基本信息

- 主办方：AMP（Accelerating Medicines Partnership）
- 任务类型：表格数据 / 医疗回归
- 评价指标：**SMAPE**（Symmetric Mean Absolute Percentage Error）
- 数据特点：小样本、多次随访、强领域特征依赖

## 数据特征

- 蛋白质特征约 227 维
- 肽段来自多次质谱测量
- 每位患者有多次 visit，带有时间结构
- 目标是预测不同阶段的疾病进展程度

## 前排方案共性

### 1. 特征工程非常关键

高分方案几乎都大量构造：

- 患者级聚合特征
- 蛋白质 / 肽段统计特征
- visit 间隔与时间演化特征
- 蛋白质与肽段之间的关系特征

### 2. 集成通常优于单模型

代表性方案常见组合：

- LightGBM
- Neural Network / MLP
- 简单平均或加权平均集成

### 3. 数据泄露控制是红线

这类医疗时序表格任务最容易在以下地方出错：

- 把同一患者不同 visit 拆进训练和验证两侧
- 在特征构造中混入未来信息
- 用测试集统计量污染训练过程

因此高分方案通常采用：

- patient-level split
- GroupKFold
- 明确按时间顺序构造特征

## 代表性经验

### 1st Place

- 强调蛋白质和肽段的聚合特征
- 使用 LightGBM + Neural Network 集成
- 对神经网络输入做专门标准化处理

### 2nd / 3rd Place

- 继续强化特征工程
- 在不同患者组或特征组之间采用差异化建模
- 保持验证方式严格一致

### 中后排值得注意的发现

- 传统 GBDT 并不总是最优
- 纯模型堆叠不如高质量特征来得稳定
- “无数据泄露”策略本身就是重要竞争力

## 实操建议

1. 先把 patient-level split 设计正确。
2. 再做蛋白质 / 肽段聚合和时间特征。
3. LightGBM 作为强 baseline，MLP 作为补充。
4. 优先做稳健集成，不要急着上复杂模型。
5. 每加一个新特征，都要重新审视是否引入 leakage。

## 关键启发

这场竞赛再次说明，在小样本医疗表格任务里：

- **特征工程** 决定下限
- **验证设计** 决定可信度
- **泄露控制** 决定上限能否成立
