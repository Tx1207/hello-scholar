# 语气指南

本文档用于约束 rebuttal 写作语气，帮助保持专业、礼貌和有说服力。

## 核心原则

### 1. 始终先感谢

每条回复最好都以感谢开头，即使你不同意 reviewer 的建议。

推荐：
- `We thank the reviewer for...`
- `We appreciate the reviewer's...`
- `We are grateful for...`

### 2. 保持尊重

即使 reviewer 误解了，也不要写：
- `The reviewer is wrong`
- `The reviewer misunderstood`

更好的写法：
- `We respectfully note that...`
- `We would like to clarify that...`

### 3. 客观陈述

使用事实、结果和证据，不要用情绪化表达。

推荐：
- `Our experiments show that...`
- `The results demonstrate...`
- `Based on our analysis...`

避免：
- `Obviously...`
- `Clearly...`
- `Everyone knows...`

### 4. 建设性回应

即使不同意，也要给出解释或替代方案，而不是生硬拒绝。

### 5. 承认不足

当 reviewer 指出真实问题时，应直接承认并说明改进。

## 常用表达

### 感谢类

- 一般感谢：`We thank the reviewer for this comment.`
- 强调价值：`We thank the reviewer for this valuable suggestion.`
- 指出错误：`We thank the reviewer for catching this.`

### 同意类

- `We agree that...`
- `We concur with the reviewer that...`
- `We acknowledge this limitation.`

### 不同意类

- `We respectfully note that...`
- `However, we chose [method] because...`
- `While [suggestion] would be valuable, it is not feasible due to...`

### 澄清类

- `We would like to clarify that...`
- `We apologize for the confusion.`
- `This is discussed in Section X (page Y).`

### 补实验类

- `We have conducted additional experiments on...`
- `The results show that...`
- `These new results have been added to...`

## 典型场景

### 审稿人完全正确

语气：直接接受，感谢指出。

### 审稿人部分正确

语气：先承认合理部分，再礼貌说明剩余部分。

### 审稿人误解论文

语气：礼貌澄清，并承认论文原文可以写得更清楚。

### 审稿人提出不可行要求

语气：解释限制，并给替代方案。

### 多位 Reviewer 意见冲突

语气：平衡双方观点，说明最终选择理由。

## 最终检查清单

### 必须包含

- [ ] 每条回复都以感谢开头
- [ ] 使用 `We` 而不是 `I`
- [ ] 给出具体位置引用
- [ ] 说明具体改动动作
- [ ] 全文语气礼貌一致

### 必须避免

- [ ] `The reviewer is wrong`
- [ ] `Obviously` / `Clearly`
- [ ] 攻击性语气
- [ ] 模糊承诺
- [ ] 忽略 reviewer comment

## 使用建议

1. 先把内容写完，再做一轮语气审查
2. 把不恰当表达替换成更专业的句式
3. 检查全篇语气是否统一
4. 最好让导师或同事再看一遍语气

目标是让 reviewer 感到：
- 他们的意见被认真对待
- 作者做了实质性改进
- 作者合作、专业、可信
