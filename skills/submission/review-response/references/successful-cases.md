# 成功 Rebuttal 案例库

本文档整理真实成功 rebuttal 的高频模式，便于模仿其结构和动作。

## 典型成功案例

### 案例 1：回应清晰度问题

高质量回应通常会：
- 承认表述不清
- 展示“原文 vs 修订文本”
- 增加算法、图示、appendix 等多种说明方式
- 明确指出修改位置

### 案例 2：回应缺失实验

高质量回应通常会：
- 感谢 reviewer 建议
- 立刻补实验
- 给出具体数值结果
- 解释为什么结果支持当前方法
- 指出新增表格 / 小节位置

### 案例 3：回应统计显著性质疑

有效回应会同时给出：
- confidence intervals
- p-value
- effect size
- 实际意义解释

不要只说“提升显著”，而不提供统计细节。

### 案例 4：回应 baseline 不足

理想做法：
- 能补的 baseline 就补
- 不能补的 baseline 说明技术上为什么不可比
- 给出具体理由，而不是一句 “超出范围”

### 案例 5：回应 overclaiming

好的做法：
- 承认 claim 太强
- 展示原句与修订句
- 新增 limitation discussion
- 用更准确语言重新表述贡献

### 案例 6：回应写作质量问题

有效回应会说明：
- 如何重构章节
- 是否补了 subsection headings
- 是否增加 notation table / appendix
- 是否修了 grammar 和 flow

## 通用成功模式总结

### 模式 1：感谢 + 行动 + 证据

```text
We thank the reviewer for [point].
We have [action].
[Evidence / result / citation]
```

### 模式 2：承认 + 修正 + 说明

```text
We agree that [issue].
We revised [location].
This addresses the concern by [explanation].
```

### 模式 3：解释 + 证据 + 引用

```text
We respectfully note that [position].
This is supported by [evidence].
```

### 模式 4：新增内容 + 位置 + 影响

```text
We have added [new content].
Location: [Section / Table / Figure].
This strengthens [claim].
```

## 避免的错误模式

### ❌ 防御性语气

不要写：
- `The reviewer is wrong`
- `The reviewer failed to notice`

### ❌ 模糊承诺

不要只说：
- `We will add more experiments`

要说清：
- 补了什么实验
- 结果如何
- 在哪里

### ❌ 忽略问题

不要只说：
- `This is beyond the scope`

更好的写法是：
- 说明为什么当前无法做
- 给出替代分析或未来计划

## 使用建议

1. 找与你当前 reviewer concern 最接近的案例
2. 借鉴其结构，而不是照抄句子
3. 每次都落到具体 evidence
4. 对所有修改说明位置
5. 保持诚实，不夸张
