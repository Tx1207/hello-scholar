# 审稿意见分类标准

本文档提供系统化的审稿意见分类方法，帮助快速识别和排序不同类型的 reviewer comments。

## 分类体系

### 1. Major Issues（重大问题）

定义：涉及方法、实验设计、结果解释或贡献有效性的核心问题。

常见特征：
- 质疑方法有效性
- 要求补关键实验或 baseline
- 指出 claim 与 evidence 不匹配
- 质疑创新性或贡献

建议策略：`Accept + Experiment` 或 `Defend`

### 2. Minor Issues（次要问题）

定义：需要澄清、补说明或小改，但不影响核心贡献。

常见特征：
- 要求解释细节
- 建议改进表达和结构
- 建议补相关工作
- 提醒图表可优化

建议策略：`Accept + Clarify`

### 3. Typos / Formatting（拼写与格式）

定义：拼写、语法、格式不一致、图表标注错误等表层问题。

建议策略：直接 `Accept`

### 4. Misunderstandings（误解）

定义：Reviewer 误读了论文，或论文已有内容不够显眼。

识别信号：
- “The authors did not...” 但实际上已经做了
- “It is unclear...” 但文中已有明确说明
- “Why don’t the authors...” 但论文已有相应内容

建议策略：`Clarify`

## 使用指南

### 分类流程

1. 第一遍快速通读所有意见
2. 给每条意见标注类型
3. 按优先级排序
4. 为每条意见匹配回复策略

### 优先级建议

高优先级：
- Major Issues
- Misunderstandings

中优先级：
- Minor Issues

低优先级：
- Typos / Formatting

### 注意事项

- 不要忽略任何意见，即使只是 typo
- 对误解也要礼貌回应，不要指责 reviewer
- 对 Major Issues 必须有实质性动作，不能只做表面澄清
