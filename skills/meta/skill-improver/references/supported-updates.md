# 支持的更新类型

## YAML Frontmatter 更新

可更新：
- `name`
- `description`
- 其他合法 frontmatter 字段

重点是：
- 保持 YAML 有效
- 不丢失未改动字段

## SKILL.md 内容更新

常见包括：
- 改成 imperative form
- 修正 trigger phrases
- 调整边界说明
- 重组 workflow

## references/ 文件更新

可新增或修改 reference sections，例如：
- 补详细说明
- 拆分主题
- 清理重复信息

## 新文件创建

可创建新：
- `examples/demo.md`
- `references/*.md`
- `scripts/*.sh`

前提是与 skill 的目标真正相关，而不是无意义占位文件。
