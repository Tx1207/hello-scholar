# Improvement Plan 格式

定义 `improvement-plan-{skill-name}.md` 的文件格式。该文件由 `skill-quality-reviewer` 生成，并由 `skill-improver` 消费。

## 文件结构

```markdown
# Skill Improvement Plan: {skill-name}

## Priority Summary
- High Priority
- Medium Priority
- Low Priority

## High Priority Improvements

### 1. [Issue Title]
**File**: SKILL.md:line:line
**Dimension**: {Dimension Name}
**Impact**: +X points

**Current**:
```yaml or markdown
...
```

**Suggested**:
```yaml or markdown
...
```
```

## 解析规则

### 优先级分区

通过这些标题识别：
- `## High Priority Improvements`
- `## Medium Priority Improvements`
- `## Low Priority Improvements`

### 条目结构

每条改进以：

```markdown
### N. [Issue Title]
```

开头，并至少包含：
- `File`
- `Dimension`
- `Impact`
- `Suggested`

对于编辑型改动，还应包含：
- `Current`

## File 字段

常见形式：
- `SKILL.md:10:15`
- `references/patterns.md`
- `examples/demo.md`

含行号表示替换区间；无行号通常表示新建文件或追加。

## 代码块格式

支持：
- `yaml`
- `markdown`
- `bash`

代码块语言决定高亮和后续校验方式。

## 校验规则

### 必需内容

- [ ] 至少包含优先级分区
- [ ] 每条改进有 `File`
- [ ] 每条改进有 `Dimension`
- [ ] 每条改进有 `Impact`
- [ ] 每条改进有 `Suggested`

### 文件路径有效性

- [ ] 路径格式正确
- [ ] 行号为整数（若存在）
- [ ] 文件扩展名和内容类型基本匹配

### 代码块有效性

- [ ] opening fence 存在
- [ ] closing fence 存在
- [ ] language identifier 合法
- [ ] suggested 内容不为空
